# 前端编码与交互规范

## 1. 目的

记录前端页面、交互、查询和防重复提交规范。

## 2. 交互规范

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| FE-UX-001 | 提交类按钮必须防抖、节流或 loading 锁定。 | 防止重复点击、重复下单或重复请求。 |
| FE-UX-002 | 异步操作必须处理 loading、空状态和错误状态。 | 不让用户面对无反馈界面。 |
| FE-UX-003 | 删除、撤销、提交等高风险操作需要确认或可恢复设计。 | 以项目既有交互模式为准。 |

## 3. 查询规范

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| FE-QRY-001 | 列表页必须有默认查询条件。 | 常见为时间范围、状态或分页限制。 |
| FE-QRY-002 | 模糊查询必须有长度限制。 | 例如 8 位以上精确查，否则不查或受控模糊查。 |
| FE-QRY-003 | 禁止一个输入框混合查询多种字段。 | 除非需求明确并定义匹配优先级。 |

## 4. Self-review 映射

| 自查项 ID | 来源规则 | 检查内容 |
|-----------|----------|----------|
| SR-FE-001 | FE-UX-001 | 提交类按钮是否防重复点击。 |
| SR-FE-002 | FE-QRY-001 | 列表页是否设置默认查询边界。 |
