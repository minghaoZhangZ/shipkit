# API契约与DTO-VO规范

## 1. 目的

记录接口入参、出参、错误响应、兼容性和 DTO / VO 使用规范。

## 2. 契约规则

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| API-001 | Request 使用 DTO 封装。 | DTO 负责输入契约和校验。 |
| API-002 | Response 使用 VO 封装。 | VO 负责前端展示契约，不暴露 Entity。 |
| API-003 | 对外字段变更必须写入数据契约设计。 | 包括字段新增、删除、语义变化和默认值。 |
| API-004 | 不做破坏性接口变更，除非用户明确确认。 | 需要兼容策略和回滚方案。 |
| API-005 | 错误响应应使用项目统一格式。 | 不向前端暴露堆栈、SQL、路径或内部类名。 |

## 3. Self-review 映射

| 自查项 ID | 来源规则 | 检查内容 |
|-----------|----------|----------|
| SR-API-001 | API-003 | API 字段变化是否进入契约文档。 |
| SR-API-002 | API-005 | 异常响应是否泄露内部细节。 |
