# SQL与数据库规范

## 1. 目的

记录 SQL、索引、DML、安全变更和回滚规范。

## 2. 索引规范

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| DB-IDX-001 | 遵循最左前缀匹配原则。 | 联合索引设计必须匹配常用查询条件。 |
| DB-IDX-002 | 单表索引建议控制在 8 个以内。 | 超出时说明必要性。 |
| DB-IDX-003 | 关联字段字符集和排序规则保持一致。 | 推荐 `utf8mb4_general_ci`，以项目实际约定为准。 |

## 3. 查询规范

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| DB-QRY-001 | 避免 `SELECT *`。 | 只查询需要的字段。 |
| DB-QRY-002 | 字段已确定唯一时，不添加无意义额外条件。 | 避免误导优化器和后续维护者。 |
| DB-QRY-003 | 避免 `Using filesort` 和 `Using temporary`。 | 复杂查询需说明执行计划。 |

## 4. 变更安全

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| DB-SAFE-001 | 严禁 `DROP TABLE IF EXISTS`。 | 数据库破坏性操作必须人工确认。 |
| DB-SAFE-002 | DML 初始化语句必须带明确主键 ID。 | 防止多环境 ID 不一致。 |
| DB-SAFE-003 | 重要变更必须提供回滚语句。 | strict 流程下必须进入回滚方案。 |

## 5. Self-review 映射

| 自查项 ID | 来源规则 | 检查内容 |
|-----------|----------|----------|
| SR-DB-001 | DB-QRY-001 | 是否存在不必要的 `SELECT *`。 |
| SR-DB-002 | DB-SAFE-003 | 数据变更是否有回滚方案。 |
