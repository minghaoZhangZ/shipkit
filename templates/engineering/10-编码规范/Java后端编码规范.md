# Java后端编码规范

## 1. 目的

记录 Java 后端代码的基础写法、数据访问、封装和注释规范。

## 2. 性能与数据访问

| 规则 ID | 规则 | 推荐写法 |
|---------|------|----------|
| JAVA-DA-001 | 禁止循环查库或循环调用 Feign / RPC。 | 循环前批量查询，使用 Map 或上下文对象缓存。 |
| JAVA-DA-002 | 避免无逻辑的 A -> B -> C 链式调用。 | B 无业务处理时由 A 直接调用 C。 |
| JAVA-DA-003 | 大列表接口必须分页。 | 使用项目既有分页对象和排序约定。 |

## 3. 代码质量

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| JAVA-CQ-001 | 单个方法尽量不超过 100 行。 | 超出时优先抽取私有方法或领域服务。 |
| JAVA-CQ-002 | 入参使用 DTO，返回前端使用 VO。 | 不直接暴露 Entity。 |
| JAVA-CQ-003 | 禁止魔法值。 | 使用常量、枚举或配置项。 |
| JAVA-CQ-004 | 使用 AssertUtils 前必须阅读定义。 | 明确是满足条件通过还是满足条件报错。 |
| JAVA-CQ-005 | 默认使用精确匹配。 | 模糊匹配必须由需求明确要求。 |

## 4. Javadoc

新增类默认包含文件头：

```java
/**
 * <p>
 * [类描述信息]
 * </p>
 *
 * @author minghao.zhang
 * @since YYYY-MM-DD
 */
```

## 5. Self-review 映射

| 自查项 ID | 来源规则 | 检查内容 |
|-----------|----------|----------|
| SR-JAVA-001 | JAVA-DA-001 | 是否存在循环查库或循环远程调用。 |
| SR-JAVA-002 | JAVA-CQ-002 | API 入参和出参是否按 DTO / VO 封装。 |
