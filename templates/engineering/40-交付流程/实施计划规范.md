# 实施计划规范

## 1. 目的

规定实施计划必须包含的内容，确保 Agent 编码前边界清晰。

## 2. 必填内容

| 规则 ID | 内容 | 说明 |
|---------|------|------|
| PLAN-001 | 总体策略和阶段拆分。 | 说明先改什么、后改什么。 |
| PLAN-002 | 允许修改的文件范围。 | 供 scope guard 和 self-review 使用。 |
| PLAN-003 | 不允许修改的文件。 | 明确防止范围蔓延。 |
| PLAN-004 | 验证命令。 | 与测试计划对应。 |
| PLAN-005 | 停止条件。 | 证据不足、范围扩大、测试无法运行时停止。 |
| PLAN-006 | 需求覆盖映射。 | 每个 Req ID 都要对应任务和测试。 |
