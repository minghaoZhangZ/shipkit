# 需求拆解规范

## 1. 目的

约束需求从原始描述拆解为可实现、可验证的 Req ID。

## 2. 拆解规则

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| REQ-001 | 原始需求必须逐条映射为 Req ID。 | 不得自行裁剪需求。 |
| REQ-002 | 每个核心功能至少有成功路径和失败路径。 | 验收标准必须可验证。 |
| REQ-003 | 标记为本次不实现的需求必须有用户确认证据。 | 否则不得跳过。 |
| REQ-004 | 涉及权限、状态、金额、库存、订单、消息、文件、外部接口时必须列边界条件。 | 默认升级 strict。 |
