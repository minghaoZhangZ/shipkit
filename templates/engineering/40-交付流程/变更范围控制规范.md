# 变更范围控制规范

## 1. 目的

防止 Agent 在实现过程中扩大范围或顺手重构。

## 2. 范围规则

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| SCOPE-001 | 只能修改实施计划允许的文件。 | 需要扩大范围时先更新计划并让用户确认。 |
| SCOPE-002 | 顺带重构默认禁止。 | 只有有证据且用户确认时允许。 |
| SCOPE-003 | 不改变已有接口行为，除非需求明确要求。 | 破坏性变更需要兼容策略。 |
| SCOPE-004 | 不为通过测试而降低门禁。 | 不关闭测试、lint、安全规则。 |
