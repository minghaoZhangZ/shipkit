# 单元测试规范

## 1. 目的

把测试作为 AI Coding 的安全网，优先用可执行测试描述业务期望。

## 2. 测试要求

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| TEST-001 | 测试必须覆盖验收标准和边界条件。 | 至少覆盖成功路径和失败路径。 |
| TEST-002 | 测试必须断言业务结果。 | 不接受只验证没有异常。 |
| TEST-003 | 修改存量逻辑前优先补保护性测试。 | 建立基线后再改代码。 |
| TEST-004 | 复杂业务规则优先 TDD。 | Red -> Green -> Refactor。 |
