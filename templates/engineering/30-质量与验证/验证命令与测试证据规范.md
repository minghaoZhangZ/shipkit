# 验证命令与测试证据规范

## 1. 目的

规定完成变更前必须提供真实验证证据。

## 2. 证据要求

| 规则 ID | 规则 | 说明 |
|---------|------|------|
| VERIFY-001 | 必须记录实际运行命令。 | 包括退出码和关键输出。 |
| VERIFY-002 | 未运行项必须说明原因。 | 不得用“应该能过”替代证据。 |
| VERIFY-003 | minimal 至少运行目标测试或最小复现命令。 | 无可运行测试时说明查找路径。 |
| VERIFY-004 | standard 运行构建、测试、lint 或 typecheck。 | 以项目已有命令为准。 |
| VERIFY-005 | strict 增加安全、数据库、回滚相关检查。 | 高风险变更不可跳过。 |
