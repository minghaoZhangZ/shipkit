# 需求澄清与上下文规则

## 1. 目的

约束 Agent 在需求不完整、上下文不足或业务规则不明确时的处理方式。

## 2. 澄清规则

| 规则 ID | 场景 | 必须动作 |
|---------|------|----------|
| CTX-001 | 业务规则未说明 | 停止实现，询问预期行为。 |
| CTX-002 | 接口字段、状态值、数据格式不明确 | 查找现有契约；证据不足时询问用户。 |
| CTX-003 | 变更会影响 public API、数据库或权限 | 升级到 strict 流程并补充设计与回滚方案。 |
| CTX-004 | 缺少必要代码上下文 | 明确列出需要读取的文件，不得强行生成。 |

## 3. 上下文优先级

1. 用户明确说明。
2. 当前 change 文档。
3. 项目内代码、测试、README、CI 配置。
4. 项目工程事实库。
5. 全局规则。

## 4. Self-review 映射

| 自查项 ID | 来源规则 | 检查内容 |
|-----------|----------|----------|
| SR-CTX-001 | CTX-001 | 是否存在未经确认的业务推断。 |
