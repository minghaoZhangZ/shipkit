# 行为准则

## 1. 目的

记录团队希望 AI Agent 和开发者在任何代码变更前都遵守的通用行为规则。

## 2. 基本铁律

| 规则 ID | 规则 | 执行方式 | 说明 |
|---------|------|----------|------|
| GEN-001 | 禁止臆造 API、库、配置或业务规则。 | enforced | 使用前必须检索代码、文档或由用户确认。 |
| GEN-002 | 需求存在歧义时先澄清，不得模糊执行。 | enforced | 包括边界条件、数据格式、异常路径和兼容性。 |
| GEN-003 | 优先复用现有工具类、组件、服务和项目模式。 | enforced | 新增能力前先搜索代码库。 |
| GEN-004 | 遵循最小改动原则。 | enforced | 不做无关重构，不扩大修改范围。 |
| GEN-005 | 修改完成必须提供验证证据。 | enforced | 写明命令、退出码、关键输出和未运行原因。 |

## 3. Self-review 映射

| 自查项 ID | 来源规则 | 检查内容 |
|-----------|----------|----------|
| SR-GEN-001 | GEN-001 | 是否存在未经验证的 API、库、配置或业务假设。 |
| SR-GEN-002 | GEN-004 | 改动是否超出实施计划允许范围。 |
