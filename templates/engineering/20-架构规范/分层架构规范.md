# 分层架构规范

## 1. 目的

记录团队已经达成共识的稳定分层架构原则。

## 2. 分层职责

| 规则 ID | 层级 | 职责 | 禁止事项 |
|---------|------|------|----------|
| ARCH-LAYER-001 | Controller | HTTP 协议、参数校验、响应码和 DTO / VO 转换。 | 编写复杂业务逻辑。 |
| ARCH-LAYER-002 | Service | 业务流程、事务边界、权限校验和领域规则。 | 返回 Entity 给前端。 |
| ARCH-LAYER-003 | Repository | 数据访问和查询封装。 | 承载业务决策。 |

## 3. Owner 决策场景

- 跨层职责发生变化。
- 新增公共基类、公共抽象或共享工具层。
- 修改被多个模块复用的 public API。
